# 디자이너 업무 요청서 (Figma)

## 문서 목적
- 본 문서는 한국 주식 AI 트레이딩 웹앱의 디자인 작업을 팀 내 전문 디자이너에게 위임하기 위한 공식 요구사항 문서다.
- 목표는 픽셀아트 에이전트 모니터링 영역을 제외한 모든 화면/상태/컴포넌트를 Figma에서 재설계하고, 토스를 참고한 독자적인 디자인 컨셉까지 확정한 후 개발 handoff 가능한 수준으로 완결하는 것이다.
- 누락 방지를 위해 화면 단위, 상태 단위, 컴포넌트 단위, 인터랙션 단위, 검수 단위 요구사항을 모두 명시한다.

---

## 1. 프로젝트 개요
### 1.1 서비스 성격
- 서비스명: Korean Trading AI Dashboard
- 핵심 기능: 종목 탐색, AI 분석, 백테스트, KIS 연동 매매, 설정 관리
- 사용 맥락: 국내 주식 투자 의사결정 지원

### 1.2 현재 IA(정보 구조)
- 단일 페이지 앱(메인 1페이지) 구조
- 좌측 패널(고정 폭) + 우측 패널(유동) 2분할
- 좌측 패널 내 탭: 분석 / 백테스트 / 매매
- 설정은 우측 슬라이드 패널(드로어), 인간 승인 모달은 중앙 오버레이

### 1.3 디자인 목표
- 사용자가 핵심 행위(분석 시작, 결과 해석, 백테스트 비교, 주문 실행)를 빠르고 안전하게 수행하도록 정보 위계를 재정의
- 금융 도메인 특성(숫자 가독성, 색상 의미, 리스크 경고)을 우선 반영
- 실서비스 적용 가능한 수준의 컴포넌트 시스템과 상태 설계를 제공

## 2. 작업 범위 정의 (중요)
## 2.1 포함 범위 (디자이너 작업 대상)
- 좌측 패널 전체
- 탭 3종 전체
  - 분석 탭
  - 백테스트 탭
  - 매매 탭(KIS)
- 공통 컴포넌트
  - 종목 검색 입력 + 자동완성
  - 인기 종목 칩
  - 주가/지표 카드
  - 탭 네비게이션
  - 실행/중단/확인 버튼 계열
  - 에러/성공 배너
- 결과 컴포넌트
  - 최종 결정 카드
  - 백테스트 결과 패널(메트릭 + 차트 영역)
- 오버레이
  - 인간 승인 모달
  - 설정 드로어
- 반응형 변형안
  - Desktop / Tablet / Mobile

## 2.2 제외 범위 (절대 제외)
- 픽셀아트 기반 에이전트 행동 모니터링 영역 전체
  - 에이전트 픽셀 캔버스
  - 픽셀 캐릭터/말풍선/레이어 애니메이션
  - 에이전트 실시간 행동 시각화 전용 요소
- 위 제외 범위의 모션/아트 스타일 재설계

## 2.3 경계 규칙
- 우측 패널 중 픽셀 모니터링 전용 영역은 디자인 변경 제안 대상이 아니다.
- 다만, 전체 페이지 레이아웃 균형을 위한 컨테이너 외곽 정렬/여백 기준은 제시 가능하다.
- 픽셀 모니터링 콘텐츠 자체의 구조/스타일/상태는 변경하지 않는다.

## 4. 디자인 시스템 필수 준수 규칙
## 4.1 도메인 색상 의미 (변경 금지)
- 국내 주식 관례 준수
  - 상승: Red 계열
  - 하락: Blue 계열
  - 보합/중립: Gray 계열
- 색상 의미가 반대로 보이는 표현 금지

## 4.2 타이포그래피
- 숫자 정보는 반드시 tabular-numeric 기준으로 정렬 가능하게 설계
- 계층
  - Display(가격/수익률)
  - Heading(섹션)
  - Body(설명)
  - Caption(보조)

## 4.3 보더/반경/그림자
- 카드/패널 중심 UI
- 과도한 글래스/블러 지양
- 금융 서비스 특성상 정보 가독성 우선

## 4.4 모션
- 상태 인지 목적의 제한적 모션만 사용
- micro-motion은 의미 기반으로만 정의
- duration/easing 명시 필수

## 4.5 접근성
- 텍스트/배경 대비 기준 충족
- 색상만으로 상태 구분하지 않기(아이콘/텍스트 병행)
- 키보드 포커스 가시성 고려

## 5.2 분석 탭 페이지 (Main_Analysis)
## 5.2.1 화면 구성 순서
1. 상단 브랜드/시장지수 영역
2. 종목 검색 + 인기 종목
3. 주가/지표 카드
4. 탭 네비게이션
5. 분석 실행/중단 버튼
6. 분석 에러 배너(조건부)
7. 파이프라인 진행 바(조건부)
8. 최종 결정 카드 또는 빈 상태
9. 시스템 구성 안내 카드
10. 하단 설정 버튼 + 투자 유의 문구

## 5.2.2 컴포넌트별 상세 요구
### A. 상단 브랜드/시장지수
- 좌측: 서비스 식별
- 우측: 지수 2개 이상 compact view
- 지수 항목
  - 지수명
  - 등락률
  - 부호(+/-)
  - 상승/하락 색상

### B. 종목 검색 + 자동완성
- 입력 상태
  - default
  - focus
  - typing
  - 결과 드롭다운 open
  - 결과 없음
- 결과 아이템 정보
  - 종목명
  - 시장(KOSPI/KOSDAQ)
  - 코드
- 인기 종목 칩
  - 선택/비선택 상태
  - hover/pressed 상태

### C. 주가/지표 카드
- 필수 표시 필드
  - 종목명/코드
  - 장 상태(개장/마감)
  - 거래량
  - 현재가
  - 변동률
  - RSI
  - 52주 범위(최저~최고 + 현재 위치)
  - MA5/MA20/MA60(없을 경우 하이픈 처리)
- 숫자 단위 표현 규칙
  - 천/만/백만 단위 가독성
- 결측값 처리 디자인
  - 데이터 없음 표시 통일
- 로딩 스켈레톤 버전 제공

### D. 탭 네비게이션
- 분석/백테스트/매매 3분할
- 분석 탭 활성 시 에이전트 활성 카운트 배지 반영 가능 구조
- active/inactive/hover/pressed 상태 정의

### E. 분석 실행 영역
- 실행 버튼 상태
  - 기본
  - hover
  - running(disabled)
- 중단 버튼
  - running일 때만 노출
- 단축키 힌트 표기 영역 포함

### F. 에러 배너
- 닫기 가능한 경고 배너
- 메시지 길이 1~2줄 가변 대응
- 재시도 유도 문구 영역 고려

### G. 파이프라인 진행 바
- 레이어 단계 3단
  - L1 데이터 수집
  - L2 토론
  - L3 결정
- 상태
  - 대기
  - 진행중
  - 완료
- 단계별 완료 수치/상태 텍스트 대응

### H. 최종 결정 카드
- 섹션 구성
  - 행동(BUY/SELL/HOLD)
  - 신뢰도 게이지
  - 인간 승인 필요 배너(조건부)
  - 에이전트 투표 분포 바
  - 결정 근거 텍스트
  - Kelly 포지션
  - 리스크 등급
  - 손절 라인
  - 진입 전략
  - 청산 전략
- 상태
  - 일반
  - 고위험 강조
  - 승인 필요 강조
- 문장 길이 가변(근거/전략) 대응

### I. 빈 상태
- 분석 전 empty
- 분석 중 empty(에이전트 n개 활성)

### J. 시스템 구성 안내
- Layer 1/2/3 설명 카드
- 정보 밀도 조절 가능한 스타일

- 주문 실패 배너

## 5.5.1 인간 승인 모달
- 오버레이 배경 + 중앙 카드
- 필수 정보
  - 액션(매수/매도/관망)
  - 종목
  - 근거 텍스트
  - 신뢰도
  - Kelly 포지션

  - 거부
  - 승인
- 상태
  - action별 색상 분기
  - 긴 텍스트 대응

## 5.5.2 설정 드로어
### 구조
- 헤더
- 스크롤 본문
- 하단 액션 바

### 섹션 1: OpenAI 연결
- API 키 입력
- 키 설정 여부 배지
- 표시/숨김 토글

### 섹션 2: LLM 모델
- 심층 모델 셀렉트
- 빠른 모델 셀렉트
- 직접 입력 케이스

## 9. 프로토타입 플로우 요구사항
- 플로우 A: 종목 검색 → 분석 시작 → 결과 카드 확인 → 매매 탭 이동
- 플로우 B: 백테스트 모드 선택 → 실행 → 결과 확인 → 재설정
- 플로우 C: 매매 주문 입력 → 확인 단계 → 주문 완료/실패
- 플로우 D: 설정 열기 → 파라미터 변경 → 저장 성공/실패
- 플로우 E: 인간 승인 모달 열림 → 승인/거부 분기

각 플로우는 클릭 가능한 프로토타입 링크 제공 필수.
