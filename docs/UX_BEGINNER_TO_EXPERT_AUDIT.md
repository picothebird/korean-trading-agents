## 0. 핵심 원칙 — "투 트랙(Two-Track) UX"

| 트랙 | 대상 | 보장해야 할 것 |
|------|------|--------------|
| **Easy Track** | 주식 입문자 / 처음 사용자 | 기본값으로 한 번도 멈추지 않고 분석 → 의사결정 → (모의)거래까지 도달 |
| **Pro Track** | 시스템 트레이더 / 고수 | 모든 숫자의 출처·계산식·가정을 한 클릭 안에서 확인. 자동매매 가드를 직접 조정 |

**원칙**:
1. **점진적 공개(Progressive Disclosure)** — 화면은 항상 "결론 → 근거 → 원본 데이터" 3계층으로. 처음엔 결론만, 펼치면 근거, 더 펼치면 원본.
2. **돈이 움직이는 모든 지점에 가드** — 실거래 모드는 색·크기·확인창 3중 차이. 모의는 마음 편히, 실거래는 의식적으로.
3. **스마트 디폴트 우선, 입력은 최후** — 현재가·잔고·성향이 있으면 그걸로 채우고, 사용자는 "수정"만.
4. **모든 의사결정에 추적 가능한 근거 표시** — `confidence: 72%`만 띄우지 말고, "왜 72인지"의 한 줄과 "더 보기" 링크.
5. **이탈 방지를 위한 흐름 연결** — 분석 끝났으면 다음 행동 제안. 빈 화면 금지.

## 2. 개선 전략 — 4단계 로드맵

각 단계는 **누적 가치(이전 단계 없이도 가치 있음) + 의존성 최소화** 원칙.

### Phase 1 — "안전 가드" (CRITICAL 우선, 1주 내) ✅ 완료
> "내 돈이 자동으로 날아가는 거 아닌가" 공포 즉시 제거.

1. ✅ **K1 — 모의/실거래 모드 시각 분리 강화** *(KisPanel 헤더 배경색 분기 + 큰 라벨)*
2. ✅ **K3 — 첫 실거래 진입 가드** *(체크리스트 모달 + 1주 한도 권장)*
3. ✅ **K5 — 주문 결과 메시지 모드 명시** *(토스트/배너 색·아이콘 통일)*
4. ✅ **A1 — AutoLoop 포지션 요약 + [일시중지] [즉시 청산] 분리 버튼** *(청산은 손익 미리보기 확인창)*
5. ✅ **A2 — 일일 손실/주문 한도 가드 UI 노출** *(설정 + 모니터링 카드, 한도 초과 자동 일시중지)*

**완료 후 사용자 변화**: 실거래 실수 가능성 거의 0. 자동매매를 켜놓고도 "언제든 멈출 수 있다"는 안심.

### Phase 2 — "흐름 연결 + 스마트 디폴트" (HIGH, 2주) ✅ 완료
> 입력 노력 절반으로. 한 화면에서 다음 행동까지.

6. ✅ **K2 + K4 — 주문 수량/가격 자동 계산** *(잔고·현재가·최대비중%로 채우고 사용자는 수정만)*
7. ✅ **D5 + P3 — 분석 결과 → 다음 행동 CTA 3종** *(모의 1주 / 백테스트 / 자동매매)*
8. ✅ **B1 — 백테스트 기간 프리셋 칩** *(분석 탭에서 진입 시 자동 기간)*
9. ✅ **B2 — 백테스트 결과 신호등 배지** *(Sharpe/Calmar/MDD 좋음·보통·나쁨)*
10. ✅ **A3 + A4 — AutoLoop Easy/Advanced 분리 + bps→%/원 환산 표시**
11. ✅ **PF1 — Portfolio 후보군 추천 시작 세트**
12. ✅ **R1 — AnalysisReport Executive Summary 카드 + 섹션 접기**
13. ✅ **P4 — 마지막 탭 위치 기억**

### Phase 3 — "온보딩 + 신뢰 구조" (1개월) — 진행중
> 처음 들어온 사람도 끝까지 따라오게. 고수는 근거를 보고 신뢰하게.

14. ✅ **P1 — 첫 진입 온보딩 위저드** *(5단계 모달 투어, Skip 가능)*
15. ✅ **K6 + S4 — KIS 토큰 발급 Stepper + 외부 가이드**
16. ✅ **D1 + D3 — 신뢰도/Kelly 산식 공개 패널** *(클릭 시 step-by-step)*
17. ✅ **B3 — 백테스트 가정 아코디언** *(슬리피지/체결지연/룩어헤드 정책 전체 노출)*
18. ✅ **R2 + R3 — 회의록 파이프라인 시각화 + 토론 스코어 공개**
19. ✅ **A6 + A7 — AutoLoop 로그 NOT_EXECUTED 사유 + 설정 반영 시점 표기**
20. ✅ **C1 + C2 + C3 — 차트 간단/상세 토글 + 색상 범례 + 사유 호버**

### Phase 4 — "심화 신뢰 시스템" (선택, 1~2개월)
> 플랫폼 신뢰도와 차별성을 만드는 깊은 기능.

24. ⬜ **AgentOffice 클릭 → 에이전트 트레이스 사이드 패널**  *(범위 외 — 사용자 제외 요청)*
25. ✅ **PortfolioLoop 상관관계/분산 경고 + 리밸런싱 정책 표기**

### Phase 4 — "심화 신뢰 시스템" (선택, 1~2개월)
> 플랫폼 신뢰도와 차별성을 만드는 깊은 기능.

## 4. 포트폴리오 기능 분리 + 사용자 라이프사이클 재설계

> **추가 감사 (2026-04-26)** — "포트폴리오"라는 단어가 현재 플랫폼에서 한 가지 의미(자동매매 루프)로만 쓰이고 있어, 실제 사용자가 주식을 다루는 사고 흐름과 어긋납니다. 본 섹션은 이를 해소하기 위한 별도 재설계 플랜입니다.

> *(에이전트 사이드 패널은 별도 개편 중이라 본 섹션 범위에서 제외)*

### 4.1 현재 상태 — 포트폴리오 관련 기능 전수조사

| 영역 | 현재 위치 | 상태 | 사용자 입장 문제 |

**구조적 결론**: 분석/거래/자동매매 엔진은 다 있지만, **사용자의 종목 라이프사이클(발견→관심→매수→보유→정리)을 한 줄로 잇는 "내 포트폴리오" 화면이 없음**. 분석은 분석 탭에, 보유는 거래 탭에, 자동매매는 포트폴리오 탭에 흩어져 있어 "내 종목"이라는 일관된 객체가 존재하지 않습니다.

### 4.2 사용자 행동 모델 vs 현재 지원

한국 개미 투자자의 5단계 사고 흐름과 현 지원 수준:

| 단계 | 사용자 행동 | 현재 지원 | 핵심 누락 |
|-----|-----------|----------|----------|
| **① 탐색 (Discovery)** | "요즘 뭐가 좋대?" | ⚠️ 인기종목 상수만 | 개인화 추천, 테마 랭킹, "왜 추천하는지" |
| **② 검토 (Research)** | 분석 → 관심 표시 | ✅ 분석 충분 / ❌ 관심종목 없음 | 분석 후 "★관심" CTA, 관심종목 화면 |
| **③ 진입 (Entry)** | 매수 + 진입 사유 기록 | ✅ 매수 가능 / ❌ 사유 미저장 | 매수 시점 분석 스냅샷 자동 보존 |
| **④ 운용 (Hold/Manage)** | 손익·신호 모니터링, 재분석 | ⚠️ 정적 보유 표시만 | 포트폴리오 헤더, 보유 상세 패널, 신호 변경 알림, 1클릭 재분석 |
| **⑤ 정리 (Exit/Rebalance)** | 익절·손절·리밸런싱 | ⚠️ 매도 가능 / ❌ 사유 기록·추천 없음 | 임계 도달 시 자동 재분석, 리밸런싱 제안 카드 |

### 4.3 분리 전략 — "Discovery" vs "My Portfolio" 명확 분리

사용자가 명시 요청한 분리는 곧 **"받아보는 화면" vs "내 것을 다루는 화면"** 의 분리입니다.

#### 새 IA(Information Architecture) — 탭 구조 개편

```
[현재] Analysis | Backtest | Trading | Portfolio (자동매매)
                                       ↓ 분리 ↓
[개편] Analysis | Backtest | Trading | Discovery | My Portfolio
                                          ↑              ↑
                                  추천·발견       내 보유·자동운용·
                                  (받아보는)        라이프사이클 추적
```

또는 **포트폴리오 묶음 사이드 그룹**으로 묶기:

```
Analysis | Backtest | Trading | 📂 포트폴리오 ▾
                                  ├─ Discovery (발견)
                                  ├─ Watchlist (관심)
                                  └─ My Portfolio (운용)
```

#### Discovery 화면 (받아보는 쪽) — 핵심 구성

```
┌─ 🌟 당신을 위한 AI 추천 (개인화 카로셀)
│   카드: [종목·현재가·등락][추천 사유 1줄][신뢰도][예상 신호]
│         [상세 보기] [★ 관심종목]
│   근거: 내 거래 이력의 섹터 선호 + 분석 이력의 신뢰 신호 패턴
│
├─ 🎯 카테고리 랭킹 ([테마][시총][모멘텀][저평가][배당])
│   각 카테고리 Top 10 + 스코어 + "왜 상위인지" 1줄
│
├─ 🤖 PortfolioLoop가 보고 있는 후보
│   현재 활성 루프의 universe_scanner 결과 노출 (이미 백엔드 보유)
│   → 사용자가 "자동 루프가 이걸 좋게 보네" 확인 후 직접 행동 가능
│
└─ 📰 시장 인사이트
    KOSPI/KOSDAQ/환율/금리 + "오늘 주목" 3줄
```

핵심: **모든 추천 카드에 "왜 추천하는가" 1줄 근거 + [★관심] [상세분석] CTA**.

#### My Portfolio 화면 (내 것을 다루는 쪽) — 핵심 구성

```
┌─ 헤더: 총자산 / 현금 / 평가손익 / 일일변동 / 누적수익률
│
├─ 보유 종목 리스트 (클릭 → 상세 패널)
│   각 행: 종목 / 수량 / 평균단가 / 현재가 / 손익 / [현재신호 ↑→↓] / [자동관리 배지]
│
├─ [선택 종목 상세 패널 (사이드 슬라이더)]
│   ├─ 매수 사유: "당시 분석 보고서 보기" 링크 + 신뢰도·날짜
│   ├─ 현재 신호: 최근 분석 요약 + [재분석 실행] 버튼
│   ├─ 손익 추세 미니 그래프 (최근 7~30일)
│   └─ 행동 제안: [보유] [추가매수] [축소] [전량매도] [추적관찰]
│         + 청산 시 손익 미리보기
│
├─ 거래 이력 타임라인 (시간순, 각 거래에 "당시 신호·신뢰도" 컨텍스트 + 분석 링크)
│
├─ 자동매매 운영 상태
│   ├─ 활성 AutoLoop 목록 (단일 종목별)
│   └─ 활성 PortfolioLoop 목록 (다종목 오케스트레이션)
│
└─ 오버뷰 액션: [+ AutoLoop] [+ PortfolioLoop] [🔄 리밸런스] [🔍 일괄 재분석]
```

#### Watchlist 화면 (Discovery → My Portfolio 사이 다리)

```
관심종목 N개 / 필터: [전체][↑상승신호][→관망][↓하강신호]
각 행: 종목 / 가격·등락 / 추가일 / 최근신호 / [분석][매수][제거]
배치 액션: [선택 종목 일괄 분석] [PortfolioLoop 후보로 추가]
```

### 4.4 라이프사이클 자동화 — 에이전트 연계 흐름

각 단계마다 어떤 에이전트/스케줄이 자동으로 도는지:

| 트리거 | 자동 실행 | 사용자에게 보이는 것 |
|--------|----------|---------------------|
| **관심종목 ★ 등록** | `watchlist` 추가 → 일간 정기분석 스케줄 등록 | "내일 아침 자동 분석 예약됨" 토스트 |
| **수동/자동 매수 체결** | `holdings_snapshots` 작성 (분석 session_id 링크) + 신호 모니터링 시작 | My Portfolio에 "매수 사유 보기" 링크 즉시 활성 |
| **보유 중 신호 변경** | 긴급 재분석 (확인용) → signal_history 기록 | 알림 카드 "삼성전자: 보유→매도 신호 (신뢰 72%) [최신 분석] [매도]" |
| **손절/익절 임계 도달** | 자동 재분석 → 신뢰도 75% 이상이면 GURU 정책에 따라 자동 청산 또는 권고 | "손절 -3% 도달, 권고: 매도" 알림 |
| **주간/월간 정기** | PortfolioLoop이 전 보유 종목 재분석 + 가중치 재산출 | "이번 주 리밸런싱 제안" 카드 (사용자 승인 필수) |

원칙:
- **자동 실행은 항상 "근거 + 되돌리기 옵션"과 함께** 노출. "왜 이렇게 됐는지" 없이 거래만 일어나면 안 됨.
- **자동 매도/청산은 GURU 정책의 명시적 허용 + 신뢰도 임계 통과 시에만**. 기본은 권고 후 승인.

### 4.5 데이터 모델 / API 추가 항목 (요약)

**새 컬렉션** (자세한 스키마는 [USER_LEVEL_DB_SCHEMA.md](USER_LEVEL_DB_SCHEMA.md)에 추가 권장):

| 컬렉션 | 목적 | 핵심 필드 |
|--------|------|-----------|
| `watchlist` | 관심종목 | user_id, ticker, source, priority, latest_signal |
| `holdings_snapshots` | 진입 근거 | user_id, ticker, entered_at, entry_analysis_session_id, entry_confidence, exited_at, exit_reason |
| `signal_history` | 신호 변경 이력 | user_id, ticker, date, previous→new_signal, analysis_session_id |
| `recommendations` | 추천 로그 | user_id, ticker, source, reason, confidence, user_action |
| `rebalancing_history` | 리밸런싱 이력 | user_id, rebalance_id, proposed_trades, executed_trades, total_pnl |

**기존 컬렉션 확장**:
- `user_trades`: `linked_analysis_session_id`, `decision_confidence`, `entry_reason`, `auto_loop_id` 추가
- `user_settings`: `portfolio_preferences` (rebalance/alert 설정), `discovery_preferences` (테마 관심사)

**새 API 엔드포인트**:
- Watchlist: `POST /api/watchlist/add`, `DELETE /api/watchlist/{ticker}`, `GET /api/watchlist`, `GET /api/watchlist/signals`
- Portfolio: `GET /api/portfolio/summary`, `GET /api/portfolio/holdings`, `GET /api/portfolio/{ticker}/reason`, `POST /api/portfolio/{ticker}/reanalyze`, `GET /api/portfolio/trades/timeline`, `GET /api/portfolio/performance`
- Discovery: `GET /api/discovery/recommendations`, `GET /api/discovery/categories`, `GET /api/discovery/portfolio-candidates`, `GET /api/market/insights`
- Signal: `GET /api/portfolio/{ticker}/signal-history`, `GET /api/portfolio/alerts`
- Rebalance: `POST /api/portfolio/rebalance/plan`, `POST /api/portfolio/rebalance/{id}/execute`, `GET /api/portfolio/rebalance/history`

### 4.6 단계적 구현 로드맵 (포트폴리오 재설계 전용)

> 이 로드맵은 본 문서 §2의 4단계 로드맵과 **병행**되며, §2 Phase 1(안전 가드) 종료 후 시작 권장.

#### **PF-Phase 1 — 라이프사이클의 뼈대** (2~3주, 최우선)
1. 백엔드: `watchlist`, `holdings_snapshots` 컬렉션 + 위 4개 endpoint(watchlist add/list/delete, holdings reason)
2. 프론트: **Watchlist 탭** 신설 (가장 단순 형태 — 리스트 + 추가/삭제 + 일괄 분석)
3. AnalysisReport / DecisionCard에 **[★ 관심종목 추가]** CTA
4. KisPanel 보유 종목 행 **클릭 가능** → HoldingDetailPanel(매수 사유 링크 + 재분석 버튼)
5. **My Portfolio 탭** 신설(또는 기존 Portfolio 탭 분할): 헤더(총자산·일변동) + 보유 리스트 + 자동매매 운영 상태

#### **PF-Phase 2 — Discovery (받아보는 화면)** (2~3주)
6. 백엔드: `recommendations` 컬렉션 + Discovery API 4종
   - 개인화 로직: user_trades 섹터 선호 + 분석 이력 신뢰 패턴 → 가중치
7. 프론트: **Discovery 탭** 신설 (4 섹션: AI 추천 카로셀 / 카테고리 랭킹 / PortfolioLoop 후보 / 시장 인사이트)
8. Discovery → Watchlist → My Portfolio 자연 연결 흐름 검증

#### **PF-Phase 3 — 신호 모니터링 + 알림** (2주)
9. 백엔드: `signal_history` 컬렉션 + 정기 분석 스케줄 (관심·보유 종목 일 3회) + 신호 변경 감지 → 알림 큐
10. 프론트: My Portfolio에 신호 변경 배지/알림 카드, [재분석] 1클릭, 손절/익절 임계 설정 UI

#### **PF-Phase 4 — 거래 이력 컨텍스트 + 성과** (1~2주)
11. 백엔드: `user_trades`에 분석 링크/신뢰도 필드 채우기 (체결 시점 자동) + portfolio/performance API
12. 프론트: 거래 이력 타임라인(각 행에 당시 신호 컨텍스트), 성과 차트(주/월/누적)

#### **PF-Phase 5 — 리밸런싱 의사결정 표면화** (1~2주)
13. 백엔드: `rebalancing_history` + plan/execute API. PortfolioLoop의 가중치 재산출을 "제안서" 형태로 노출
14. 프론트: My Portfolio 상단에 "이번 주 리밸런싱 제안" 카드(승인 전 미리보기 + 수동 편집)

### 4.7 즉시 가능한 단발 액션 (1~3시간 내)

§2의 Quick Wins와 별개로, 포트폴리오 라이프사이클 시각에서 **오늘 바로 손댈 수 있는 작은 개선들**:

1. **AnalysisReport / DecisionCard에 [★ 관심종목] CTA 버튼 자리만 우선 추가** (실제 저장은 localStorage 임시 사용 → Phase 1에서 백엔드 연결)
2. **KisPanel 보유 종목 행을 클릭 가능하게** → 모달로 종목명·수량·평가손익만 우선 표시 (Phase 1에서 분석 링크 추가)
3. **PortfolioLoopPanel / AutoLoopPanel 헤더에 "이건 무엇 하는 패널인가" 1줄 설명 배너** 추가 → 두 패널 책임 혼동 즉시 해소
4. **PortfolioLoopPanel 활동 로그에 카테고리 필터** ([전체][후보발견][분석][거래][리밸런스]) 추가
5. **KisPanel에 "최근 거래 3건" 미니 리스트** 추가 (이미 `getMyTrades()` 호출 가능) → 거래 이력 surface 부재 즉시 완화
6. **page.tsx 탭 설명 한 줄에 "포트폴리오 = 다종목 자동운용 / 매매 = 단일 수동주문"** 차이 명시 (탭 설명 텍스트 보완)
7. **활성 PortfolioLoop의 universe 후보 N개를 패널 상단에 노출**(이미 백엔드 데이터 보유) → Discovery 화면 부재 임시 완화

### 4.8 분리 후 사용자 흐름 검증 (E2E)

개편 완료 시 사용자 시나리오:

> "삼성전자 같은 거 사보고 싶은데 뭐가 좋을지 모르겠어"
> 1. **Discovery 탭** 진입 → AI 추천 카로셀에서 "삼성전자 신뢰 78%" 카드 발견
> 2. [상세보기] → Analysis 탭으로 자동 이동, 분석 결과 확인
> 3. 결과 만족 → [★ 관심종목 추가]
> 4. **Watchlist 탭** → 며칠 모니터링하며 신호 안정 확인
> 5. [매수] → KisPanel로 자동 이동(수량·가격 자동 prefill)
> 6. 체결 즉시 **My Portfolio**에 등장 + 매수 사유 자동 저장
> 7. 며칠 후 신호가 SELL로 바뀌면 알림 → 1클릭으로 [최신 분석 보기] → 결정
> 8. 매도 후 거래 이력 타임라인에 "왜 샀고 왜 팔았는지" 영구 보존

이 흐름이 **한 번도 끊기지 않고** 흘러가는 것이 본 분리의 목표입니다.
